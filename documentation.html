Description

A full-stack real estate application that allows users to browse, list, and purchase properties. Built with Django REST Framework backend and React frontend with Google Maps integration.

Tech Stack

Backend

Django, Django REST Framework, SQLite, JWT Authentication

Frontend

React, Vite, Tailwind CSS, React Router

Maps

Google Maps JavaScript API, Places API

Authentication

JWT (JSON Web Tokens)

Key Features

  • User authentication and authorization
  • Property listing and management
  • Interactive maps with property markers
  • Property search and filtering
  • Image upload for properties
  • Responsive design
  • Purchase functionality
  • User profiles and property management

General Project Questions

Q: Can you walk me through the architecture of your real estate application?

A: The application follows a client-server architecture with a React frontend and Django REST API backend. The frontend handles user interactions and UI rendering, while the backend manages data persistence, authentication, and business logic. Google Maps API is integrated for location services, and JWT tokens handle authentication.

Q: What technologies did you choose and why?

A:
  • Django REST Framework: Robust, secure, and rapid API development
  • React: Component-based UI with excellent ecosystem
  • JWT: Stateless authentication suitable for SPAs
  • Google Maps: Industry-standard mapping solution
  • Tailwind CSS: Utility-first CSS for rapid UI development

Q: How did you handle state management in the React application?

A: I used React Context API for global authentication state and local useState hooks for component-specific state. This approach keeps the application simple while providing necessary state sharing for user authentication across components.

Backend-Specific Questions

Q: Explain your Django models and their relationships.

A: I have two main models:
  • User: Custom user model extending AbstractUser with additional fields
  • Property: Contains property details with a foreign key to User (owner)
  • Relationship: One-to-many (User can own multiple properties)

Q: How did you implement authentication and authorization?

A:
  • Authentication: JWT tokens with djangorestframework-simplejwt
  • Authorization: Custom permissions ensuring users can only modify their own properties
  • Token Management: Automatic refresh mechanism on the frontend

Q: How do you handle file uploads for property images?

A: Django's ImageField with custom upload path to 'properties/' directory. Files are validated for image format and served through Django's media URL during development.

Frontend-Specific Questions

Q: How did you structure your React components?

A:
  • Pages: Route-level components (Home, Properties, Profile)
  • Components: Reusable UI components (Header, Footer, Maps)
  • Contexts: Global state management (AuthContext)
  • Services: API communication layer

Q: How did you integrate Google Maps?

A:
  • Maps JavaScript API: For interactive maps
  • Places API: For address autocomplete
  • Custom Markers: SVG-based markers showing property status
  • Info Windows: Property details on marker click

Technical Deep-Dive Questions

Q: How would you optimize the application for performance?

A:

Q: How would you handle a scenario where Google Maps API fails?

A: Implemented graceful degradation:

  • Manual Input: Allow manual address entry
  • Error Messages: Clear feedback to users
  • Retry Logic: Attempt to reload maps after failure
